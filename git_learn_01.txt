1.安装状态初始化：
 设置个人名称与邮件：
 git config --global user.name "runoob"
 git config --global user.email test@runoob.com

 设置默认文本编辑器：
 git config --global core.editor "code --wait"

 差异分析（解决合并冲突分析）：
 git config --global merge.tool vimdiff

 查看配置：
 $ git config --list
 http.postbuffer=2M
 user.name=runoob
 user.email=test@runoob.com


2.git工作区域及相互关系：

工作区：电脑内看到的目录，下载项目或在此进行代码修改的目录，执行命令（add）会提交至暂存区，暂存区目录树更新

暂存区：存放在工作区隐藏目录.git下的index文件，暂存工作区的每项修改以待提交至仓库

版本库：隐藏目录.git，暂存区提交（commit）后，版本库内对应目录树会更新 


其他：
注：HEAD	为版本库中某一分支的目录
git reset HEAD	：暂存区目录树重写，被对应版本库中目录替换。
git rm --cached <file>：直接从暂存区删除文件
git checkout -- <file>（危）：暂存区文件替换相应工作区文件，未提交至暂存区的文件会被替换
git checkout .   ：全部替换
git checkout HEAD：暂存区目录树重写，替换为暂存区以及工作区文件
git checkout HEAD <file>：全部替换
 
一般工作流程：
从远程仓库克隆项目至本地
创建新分支
工作目录下进行代码编辑
暂存文件以待提交
全部修改完毕后提交至本地仓库
（从远程仓库拉1取最新的项目避免冲突）
推送至远程仓库
